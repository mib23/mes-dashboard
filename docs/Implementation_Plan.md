# 羊毛衫厂 MES 数据大屏项目开发实施计划

## 1. 项目概述

本项目旨在为羊毛衫生产工厂开发一套实时、高效的数据可视化大屏系统。系统将覆盖工厂总览、编织车间、套口与缝合车间、后整车间等关键业务场景，实现生产进度、质量控制、设备状态的透明化与精细化管理。

本计划涵盖了从需求确认、架构设计、开发联调到最终交付试运行的完整实施生命周期。

## 2. 实施周期与里程碑

本项目预计总周期为 **10-12周**，分为以下五个核心阶段：

### 阶段一：需求调研与原型设计（第1-2周）

**目标**：明确业务边界，锁定系统原型及接口规范。

- **W1.1**：召开项目启动会，确定双方项目组成员及沟通机制。
- **W1.2**：深入编织、套口、洗水等车间进行实地调研，确认核心业务指标（如横机 OEE 计算规则、套口工序瓶颈判定逻辑）。
- **W2.1**：输出并确认高保真 UI/UX 设计图（含大屏布局、配色、数据交互动效）。
- **W2.2**：与工厂现有 ERP/MES/IoT 供应商对接，制定详细的 API 接口文档（RESTful / WebSocket / MQTT）。

### 阶段二：系统架构设计与环境搭建（第3周）

**目标**：技术选型、数据库设计及开发环境准备。

- **W3.1**：确定前后端技术栈（前端：React + ECharts；后端：Node.js + NestJS；数据库：MySQL + Redis）。
- **W3.2**：搭建开发、测试及预发布服务器环境。
- **W3.3**：设计数据库表结构，建立大屏实时数据的缓存与汇聚机制（如 Redis 缓存高频刷新的产量数据）。

### 阶段三：核心功能开发（第4-7周）

**目标**：完成前后端编码，实现各场景大屏功能。

- **W4**：**数据底座与接口联调**
  - 开发后端数据拉取与聚合定时任务。
  - 搭建 WebSocket 服务端，实现核心报警数据的实时推送。
- **W5**：**工厂总大屏与编织车间大屏开发**
  - 总大屏：完成全厂生产进度、核心设备（电脑横机等）状态总览。
  - 编织大屏：实现电脑横机矩阵状态图、断纱/漏针实时预警、OEE 统计开发。
- **W6**：**套口与缝合、后整车间大屏开发**
  - 套口缝合大屏：开发在制品（WIP）滞留预警逻辑、员工效能排行榜。
  - 后整大屏：完成洗水/烘干机工艺参数监控、订单齐套率看板开发。
- **W7**：**前端动效与自测**
  - 添加大屏的入场动画、数据轮播、报警闪烁特效。
  - 开展前后端联合调试与内部 Bug 修复。

### 阶段四：硬件部署与现场联调测试（第8-9周）

**目标**：软硬件集成，确保现场实际数据流转顺畅且展示正确。

- **W8.1**：车间大屏硬件（智能电视/LED 屏/商用显示器）及工业级主机的采购、安装与网络布线。
- **W8.2**：系统部署至生产环境，进行真实环境下的网络连通性与性能压力测试。
- **W9.1**：**现场数据比对**：将大屏数据与 MES 系统报表、车间实际生产情况进行核对（如：横机实时停机状态是否与大屏秒级同步）。
- **W9.2**：系统安全与异常演练（断网重连、宕机恢复、内存泄漏排查）。

### 阶段五：试运行与项目交付（第10-12周）

**目标**：系统平稳上线，完成用户培训与最终验收。

- **W10**：系统开启试运行（并行期）。收集厂长、车间主任等核心用户的反馈并进行微调。
- **W11**：编制《大屏系统操作手册》与《系统维护指南》，为车间组长及 IT 运维人员提供现场培训。
- **W12**：召开项目验收会议，签署验收报告，正式转入后期运维阶段。

## 3. 风险管理计划

| 风险类别     | 风险描述                                                                | 应对策略                                                                                |
| :----------- | :---------------------------------------------------------------------- | :-------------------------------------------------------------------------------------- |
| **接口风险** | 现有 MES 或 IoT 设备提供数据延迟或接口文档不全                          | 在第一阶段提前介入接口摸底；针对不支持推送的接口，采用后端高频轮询+缓存策略             |
| **网络风险** | 车间环境复杂（特别是编织、洗水车间），可能存在 Wi-Fi 信号衰减或网络波动 | 建议大屏设备优先采用**有线网络（千兆以太网）**接入；前端增加断网提示及自动重连机制      |
| **需求变更** | 试运行期间车间管理层提出新增考核指标或更改计算逻辑                      | 实行严格的需求变更流程（CR）；系统设计时预留可配置的数据指标项                          |
| **性能风险** | 大屏长期（7x24小时）运行导致浏览器内存泄漏卡顿                          | 前端代码严格管理定时器与图表实例销毁；部署自动刷新脚本（如每天凌晨2点自动刷新一次页面） |

## 4. 资源需求

- **开发团队**：项目经理 1 人，产品/UI 1 人，前端开发 2 人，后端开发 1-2 人，测试工程师 1 人。
- **硬件设施**：各车间对应尺寸的高亮度大屏显示器、无风扇工业控制计算机、挂墙支架。
- **工厂配合**：需 MES 系统供应商与 IoT 设备网络管理员在阶段一、阶段四全力配合接口开放与联调。
