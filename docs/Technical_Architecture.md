# 羊毛衫厂 MES 数据大屏项目技术架构文档

## 1. 架构概述
本项目采用前后端分离的现代化 Web 架构。针对大屏系统对高频数据刷新、多设备适配以及长期稳定运行的严格要求，我们在前端选用 **React** 作为核心视图层，后端采用 **Node.js (NestJS)** 作为 API 与实时消息网关，结合 **MySQL** 与 **Redis** 构建稳定高效的数据底座。

## 2. 核心技术栈选型

### 2.1 前端技术栈 (Frontend)
- **核心框架**：**React 18** (基于 Hooks 与函数式组件构建，具备优秀的并发渲染能力)
- **开发语言**：**TypeScript** (提供严格的类型检查，降低企业级项目维护成本)
- **可视化图表**：**ECharts** (处理大量数据渲染性能优异，支持复杂的工业图表及动效，替代体积较大的 DataV)
- **状态管理**：**Zustand** (轻量级、无样板代码，适合大屏多个组件间共享数据，如全局时间、车间筛选状态)
- **实时通信**：**Socket.io-client** (处理 WebSocket 连接、心跳保活与断线重连)
- **构建工具**：**Vite** (提供极速的冷启动和热更新，提升开发体验)
- **样式方案**：**Tailwind CSS** (原子化 CSS，便于快速构建高对比度的大屏暗色 UI)

### 2.2 后端技术栈 (Backend)
- **核心环境**：**Node.js** (事件驱动、非阻塞 I/O，极其适合处理高并发的实时数据推送场景)
- **Web 框架**：**NestJS** (提供企业级的架构规范，内置依赖注入、模块化设计，适合长期维护)
- **开发语言**：**TypeScript** (前后端统一语言与类型定义，如公用 DTO 接口)
- **实时通信**：**Socket.io** (建立与前端的 WebSocket 通道，支持房间机制/Rooms，可按不同车间大屏推送特定数据)
- **ORM 框架**：**Prisma** 或 **TypeORM** (安全、高效地进行数据库操作)
- **任务调度**：**@nestjs/schedule** (用于定时向 MES 或 IoT 系统轮询拉取最新数据)

### 2.3 数据库与缓存层 (Database & Cache)
- **关系型数据库**：**MySQL 8.0**
  - 存储大屏基础配置、设备台账映射关系、历史报警记录等持久化业务数据。
- **内存数据库/缓存**：**Redis 7.0**
  - **高频数据缓存**：缓存横机实时状态、实时产量，避免频繁击穿 MES 数据库。
  - **消息发布/订阅**：作为后台任务与 WebSocket 网关之间的消息总线（Pub/Sub）。

## 3. 系统架构图设计

```text
[ 大屏前端终端 (React + ECharts) ]
   | (HTTP / WebSocket)
   v
[ Nginx (反向代理 / 负载均衡) ]
   |
   +---> [ Node.js / NestJS 后端集群 ]
           |-- 认证与权限模块
           |-- WebSocket 消息网关 (Socket.io)
           |-- 定时任务调度器 (Cron Jobs)
           |-- 数据适配器层 (Adapter)
           |
           +---> [ Redis ] (高频状态缓存、实时指标计算)
           +---> [ MySQL ] (基础配置、历史归档数据)
           |
           v
[ 工厂内部系统与设备 ]
   |-- ERP 系统 (订单数据、物料计划 - RESTful API)
   |-- MES 系统 (工单进度、质检报告 - RESTful API / DB 视图)
   |-- IoT 平台 (电脑横机、套口机实时运行状态 - MQTT / TCP)
```

## 4. 核心机制设计

### 4.1 实时数据推送机制
1. **数据拉取**：Node.js 后端通过定时任务（如每 5 秒）调用 MES/IoT 接口，或通过 MQTT 订阅直接接收设备状态。
2. **数据清洗与缓存**：后端计算 OEE、工序瓶颈等衍生指标，更新到 Redis 中。
3. **定向推送**：前端大屏连接时加入特定的 `Room`（如 `room_knitting` 编织车间）。当 Redis 中该车间的数据发生变更时，Socket.io 仅向该 `Room` 推送增量数据，降低前端渲染压力。

### 4.2 大屏稳定性保障机制
- **断网自动重连**：Socket.io 自带重连机制，前端结合 Zustand 监听网络状态，断网时显示“离线警示”，恢复后自动拉取全量数据。
- **内存防泄漏**：React `useEffect` 中严格销毁 ECharts 实例及清除定时器。前端设置定时强制刷新（例如每天凌晨 02:00 使用 `window.location.reload()` 释放浏览器内存）。
- **进程守护**：Node.js 后端采用 **PM2** 进行进程管理，遇到未捕获异常崩溃时可秒级自动重启。

## 5. 部署方案
- **前端部署**：通过 Vite 打包出静态文件，部署在 **Nginx** 容器中。
- **后端部署**：通过 **Docker** 容器化部署 Node.js 应用，便于环境隔离与快速迁移。
- **网络拓扑**：服务器部署于工厂内网（Local Server），前端大屏终端通过千兆有线局域网访问 Nginx IP，保证数据传输极低延迟。
